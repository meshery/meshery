## What's New
Meshery v1.0.0 is generally available. For the full announcement, see [Meshery v1.0 is Generally Available](https://meshery.io/blog/meshery-v1.0-general-availability/).

## 🔤 General
- Refresh generated model artifacts and sync release data into the Hugo-based docs experience
- Remove remaining Jekyll-era references, align docs with canonical sources, and update shortcode usage
- Update Meshery server and mesheryctl error-code references
- Pin and bump schema dependencies as part of release stabilization

## ⌨️ Meshery CLI
- Add mesheryctl reference commands to the docs experience
- Improve relationship list and search commands with better output, ID visibility, pagination validation, and case-insensitive handling
- Promote the `relationship` command out of experimental status and add related reference documentation
- Refactor design onboard/offboard flows into deploy/undeploy commands and tighten validation, error handling, and golden tests
- Improve CLI logger usage, validator behavior, and file-path error messaging

## 🖥 Meshery UI
- Fix the broken hamburger menu and improve sidebar toggle accessibility
- Improve design actions in table view and refine the Kubernetes cluster status tooltip
- Enhance validation-form loading states and fix the validate-design schema validator flow
- Restore RJSF aliases for the validate extension flow and fix the extension nav drawer behavior

## 📖 Documentation
- Publish Meshery release notes in the docs site
- Update Meshery Docs CLI reference content and REST API documentation
- Update contributing documentation for the Hugo docs migration
- Sync test reports and release metadata into the docs experience

## 🧰 Maintenance
- Fix provider-ui ESLint workflow instability and align it with a stable lint configuration
- Remove trailing experimental markers and clean up release-related docs content
- Apply small repository hygiene fixes, including `.gitignore` updates
